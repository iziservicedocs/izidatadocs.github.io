    概述

    提供如身份证实名认证、人脸比对识别、个人/企业征信等API接口。

    IZI服务后台需要通过使用Access Key / Secret Key加密的方法来验证某个请求的发送者身份。Access Key（AK）用于标示用户，Secret Key（SK）是用户用于加密认证字符串和IZI服务用来验证认证字符串的密钥，其中SK必须保密，只有用户和IZI知道。

    SDK初始化，需要用户先申请到属于自己的AK和SK。

    请确保将YOUR_ACCESS_KEY和YOUR_SECRET_KEY替换为你自己的AK和SK。

    请确保将YOUR_ACCESS_KEY和YOUR_SECRET_KEY替换为你自己的AK和SK。

    身份认证

    身份证＋姓名校验身份。

    HTTP请求

    POST http://api.izi.credit/api/v1/identitycheck

    请求参数

    参数 描述
    name 可选，字符串，认证人员的姓名
    id 字符串，认证人员的身份证号NIK，格式为16位阿拉伯数字

    返回数据格式

    参数 描述
    status 状态码
    message id：身份证号码，NIK
    name：姓名
    province：省份
    city：城市
    district：区县
    village：村镇

    状态码说明

    code 说明
    OK 查找成功，库中存在查询的NIK，且姓名比对一致
    NOT 查找成功，库中存在查询的NIK，但姓名比对不一致
    PERSON_NOT_FOUND 我们库中查不到此NIK
    INVALID_ID_NUMBER 输入的NIK不合法
    RETRY_LATER 系统错误，请稍后重试

    人脸比对

    分析两张人脸属于同一个人的可能性大小。使用我们的人脸比对技术，您会得到一个置信度分数以及我们关于是否是同一个人的判断。

    代码示例：

    import izi

api = izi.client('YOUR_ACCESS_KEY', 'YOUR_SECRET_KEY')

url = "http://api.izi.credit/api/v1/facecompare"
response = api.request(url, data)
print(response)

    HTTP请求

    POST http://api.izi.credit/api/v1/facecompare

    请求参数

    参数 描述
    img1 字符串，base64编码的第一张图片
    img2 字符串，base64编码的第二张图片

    返回数据格式

    参数 描述
    status 状态码
    message similarity：相似度得分，0~100
    result：判断结论

    状态码说明

    code 说明
    OK 比对状态正常，message.similarity为相似度得分
    INVALID_REQUEST 请求参数不正确
    IMAGE_INVALID_SIZE 图片大小超过2M
